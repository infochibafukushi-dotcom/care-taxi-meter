# 05_運行状態仕様

走行中、待機中、院内付き添い、案件終了。

## 事前確定Mの状態遷移

### reservation-v4 の `meterRunStatus`

| 状態 | 意味 |
| --- | --- |
| `not_started` | 運行未開始 |
| `in_progress` | 運行中（`start-fixed-fare` 後） |
| `completed` | 運行完了（`complete-fixed-fare` 後） |

### メーターアプリ側の流れ

1. 予約詳細（`/reservations/:reservationId`）
2. `start-fixed-fare`（「事前確定Mで開始」）
3. fixed CasePage（車両選択 → 運行画面）
4. 運行開始 → 走行 → 運行終了
5. 精算保存（`caseRecords`）
6. `complete-fixed-fare`
7. 予約詳細へ戻る（「事前確定M 完了」）

- `complete-fixed-fare` は **`caseRecords` 保存成功後**にのみ呼び出す。
- `complete-fixed-fare` 失敗時は `activeTripSnapshot` / `reservationTripContext` を削除せず、再試行可能な状態を維持する。

詳細は [`08_本番運用仕様.md` §15](./08_本番運用仕様.md) を参照。
